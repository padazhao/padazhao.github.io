Master Admission System

The architecture design of the master's admissions system follows the standard three-tier architecture model, including the presentation layer, service layer, and persistence layer. It uses SpringBoot as the framework, Hibernate as the object-relational mapping (ORM), and Thymeleaf as the template engine. The system follows the Model View Controller (MVC) pattern, which is used to organize code in a way that separates concerns and improves maintainability and extensibility. The view component uses Thymeleaf as the template engine to render HTML pages. The controller component handles HTTP requests and maps them to the corresponding controllers and methods. The controller layer handles user requests and RESTful endpoints to accept and process client requests. It sends requests to the corresponding services for processing and coordinates the flow of business logic. The RestController class is also responsible for returning the response to the view after processing is completed.

Classification of Biochip Data from Video Captures of Changing Spot Values (Bachelor Thesis)

By building an Android application on the mobile terminal, a video of the antibiotics in biological experiments changing over time is placed in it. The application then slices the video into several frames of pictures, and then calculates the RGB values in each well plate through image recognition. The average RGB value in each well plate represents the well plate, and then uses a machine learning classification algorithm to summarize the changes in the color point values in the well plate to classify the types of antibiotics. Finally, the information is visualized and XAI is used to explain the logic of the final result selection process.

Link: https://github.com/padazhao/Final_Year_Project_for_biochips_video_captures
